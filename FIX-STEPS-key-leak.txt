WHAT HAPPENED
------------------------------------------------------------------------
Your GitHub Actions "Daily Content Sync" has been failing since ~July 1.
The workflow reports "Success" (because each content step has
continue-on-error: true), but ALL 5 content scripts throw this error:

    403 - "Your API key was reported as leaked. Please use another API key."
    (PERMISSION_DENIED)

Google PERMANENTLY DISABLED your Gemini API key because it detected the
key published publicly. A disabled/leaked key can NEVER be re-enabled.
It must be replaced with a brand-new key.

HOW IT LEAKED (two vectors)
------------------------------------------------------------------------
1. PUBLIC GIT HISTORY: your .env file (with GEMINI_API_KEY=... and other
   secrets) was committed in many past commits. After you made the repo
   public, Google's secret scanner found the key and killed it.
   (.env is now gitignored, but old commits still contain the value.)

2. BROWSER BUNDLE (ongoing risk): src/pages/FengShuiRoomAnalyzer.tsx
   calls Gemini directly from the browser using VITE_GEMINI_API_KEY.
   Any VITE_-prefixed variable is baked into the shipped JavaScript, so
   ANY visitor can read it in DevTools. NOTE: GEMINI_API_KEY and
   VITE_GEMINI_API_KEY are the SAME value, so one leak kills both.

========================================================================
1. Go to  https://aistudio.google.com/apikey
   (sign in with the same Google account you used before)
2. First, DELETE the old/leaked key: find it in the list, click the
   trash / delete icon. (It's dead anyway; removing it is cleanup.)
3. Click "Create API key".
4. Choose a project (or create one). Click "Create API key in new/existing
   project".
5. Copy the new key (looks like: AIza................................).
   Keep it somewhere safe TEMPORARILY - do NOT paste it into any file
   that gets committed to git.

========================================================================
 PART B - WHERE TO PUT THE NEW KEY (3 places)
========================================================================
The cron job (GitHub Actions) and the website (Vercel) read the key from
DIFFERENT places. Update all three:

1) GITHUB ACTIONS SECRET  <-- this is what fixes your failing cron
   - Repo -> Settings -> Secrets and variables -> Actions
   - Click GEMINI_API_KEY -> Update -> paste new key -> Save.

2) VERCEL ENVIRONMENT VARIABLE  <-- fixes the deployed website
   - Vercel -> my-fengshui-calculator -> Settings -> Environment Variables
   - Update GEMINI_API_KEY with the new value.
   - (If VITE_GEMINI_API_KEY exists in Vercel, update it too - but see
     PART D: this one is the leak risk and should eventually be removed.)
   - Redeploy the project so the new value takes effect.

3) LOCAL FILE  <-- fixes running the scripts on your own machine
   IMPORTANT: the sync-*.js scripts use dotenv.config(), which reads
   ".env" (NOT ".env.local"). So for local runs you must update the
   GEMINI_API_KEY line in ".env".
   .env.local is only used by Vite (the dev website), so update the
   VITE_GEMINI_API_KEY line there if you use the room analyzer locally.
